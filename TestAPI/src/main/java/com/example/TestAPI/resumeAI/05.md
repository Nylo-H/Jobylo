## Goal
- Backend Spring Boot (Java 17) d'une marketplace "JobyloBackend" avec messagerie temps réel, paiement escrow simulé, vérification KYC, et audit d'actions.

## Constraints & Preferences
- Architecture en couches : interfaces service + implémentations concrètes, DTO records, contrôleurs REST, WebSocket STOMP
- Clean code, pattern existant respecté
- Paiement 100% simulé (zéro frais, zéro intégration bancaire) avec gateway prêt à basculer vers CinetPay/NotchPay
- Commission plateforme configurable (5% par défaut)
- KYC obligatoire pour actions sensibles (messagerie, paiement) mais pas pour navigation
- Fichiers uploadés stockés sur le disque serveur, servis en statique

## Key Decisions
- **Entité Conversation plutôt qu'UUID volant** : évite les doublons (contrainte UNIQUE), ne dépend pas du premier message, performances optimisées
- **Participants ordonnés par UUID** : participant1 = UUID le plus petit, participant2 = le plus grand (ordre déterministe pour la contrainte UNIQUE)
- **KycGuard comme @Component** : réutilisable dans tous les services sensibles, message d'erreur adapté au status (null/PENDING/REJECTED)
- **DemoPaymentGateway séparé** : interface PaymentGateway pour permettre le branchement futur de CinetPay/NotchPay sans modifier le métier
- **Fichiers uploadés sur disque** : solution MVP sans dépendance externe (S3/Cloudinary), prêt à évoluer
- **Filtre status backend + frontend** : le backend filtre pour l'efficacité, le frontend garde la main pour des filtres additionnels
